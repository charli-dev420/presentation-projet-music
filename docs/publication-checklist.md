# Publication Checklist / Checklist publication

[FR](#francais) | [EN](#english)

## Francais

Avant d'ajouter une image, preuve ou document public:

- la source est publique ou explicitement autorisee;
- aucun chemin local, nom utilisateur, device ID, token, endpoint, run ID ou log n'est visible;
- aucune capture QA interne brute n'est publiee;
- chaque claim a surface, date, statut et limite;
- les binaires, APK, VST, installateurs, presets, audio, CSV et JSON bruts sont exclus;
- les details storage, paiement, admin et publication restent hors vitrine;
- l'image a ete recadree/compressee pour GitHub;
- le document indique ce que la preuve ne prouve pas.

## English

Before publishing any image, proof, or document, ensure the source is public or cleared, no private identifiers are visible, raw QA artifacts are excluded, each claim has a date/status/limit, and binaries/builds/presets/audio/raw logs are never added.
