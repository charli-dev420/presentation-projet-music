# Evidence Pack / Pack de preuves

[EN](#english) | [FR](#francais)

## English

### Purpose

This pack turns the music repository into a reviewable project dossier. It links the product story to written evidence, QA vocabulary, visual material, and practical next steps.

### DAW Core Signals

DAW Core is consistently presented as the main project: a browser-based **web local-first DAW**. The evidence lane centers on `.dw` continuity, browser/desktop workflow, Android beta testing, and release wording tied to current validation.

The current proof story is strongest when it stays small: one project, one target surface, one expected behavior, one written result. That format makes the workflow testable without turning the repo into an implementation dump.

### Unicor SoundEngine Signals

Unicor SoundEngine is presented as the separate Synthé / FX / VST line. The useful evidence is not "how many repos exist"; it is whether the instrument family, FX families, catalog, manuals, visuals, and audition material form a product line a user can understand.

Synthé has a seven-instrument structure. VST distribution has showcase images and manual references. FX has treatment families that can be prioritized by musical value.

Choose the surface first: DAW Core browser, Android beta, `.dw` continuity, Synthé, FX, VST catalog, or documentation. Then choose one scenario, one expected outcome, and one written result. The output should be concrete enough that another person can repeat or challenge it.

## Francais

### Objectif

Ce pack transforme le repo musique en dossier projet reviewable. Il relie l'histoire produit aux preuves ecrites, au vocabulaire QA, aux visuels et aux prochaines actions pratiques.

### Signaux DAW Core

DAW Core est presente de facon coherente comme le projet principal: un **web local-first DAW** dans le navigateur. L'axe preuve se concentre sur la continuite `.dw`, le workflow navigateur/desktop, les tests beta Android et une formulation release liee a une validation courante.

L'histoire de preuve est plus forte quand elle reste petite: un projet, une surface cible, un comportement attendu, un resultat ecrit. Ce format rend le workflow testable sans transformer le repo en dump d'implementation.

### Signaux Unicor SoundEngine

Unicor SoundEngine est presentee comme la ligne Synthé / FX / VST separee. La preuve utile n'est pas "combien de repos existent"; c'est de savoir si la famille instrumentale, les familles FX, le catalogue, les manuels, les visuels et l'audition forment une ligne produit comprehensible par un utilisateur.

Synthé a une structure de sept instruments. La distribution VST a des images vitrine et references manuels. FX a des familles de traitement priorisables par valeur musicale.
