<h2 id="english">🇬🇧 English</h2>

This page lists the most useful feedback, review and collaboration needs around the public music-tech projects.

The goal is not to ask for generic help. The useful needs are concrete: test a workflow, listen to an output, review a document, check a claim or identify the next proof gap.

---

## DAWWW-CORE

Useful DAWWW-CORE feedback can focus on:

- `.dw` project creation;
- save and reopen behavior;
- recovery scenarios;
- audio export behavior;
- Web Audio playback behavior;
- arranger / sequencer / piano roll workflow;
- mixer and effect behavior;
- Android testing boundaries;
- documentation clarity.

The most useful report describes what was tested, what happened, what failed or felt unclear, and what should be checked next.

---

## UnicorSoundEngine

Useful UnicorSoundEngine feedback can focus on:

- listening quality;
- instrument roles;
- effect usefulness;
- preset naming and usability;
- manual readability;
- product-page clarity;
- packaging state;
- download flow;
- DAW compatibility for native plugins.

---

## FX Line

Useful FX review can focus on:

- before / after examples;
- control naming;
- preset usefulness;
- UI clarity;
- build status;
- packaging status;
- DAW compatibility;
- which effects should be prioritized first.

---

## Documentation

Useful documentation review can focus on:

- whether the project is understandable to an external reader;
- whether DAWWW-CORE and UnicorSoundEngine are clearly separated;
- whether any claim sounds too strong;
- whether the public proof boundaries are clear;
- whether the reading path is short enough;
- whether pages feel like public documentation rather than internal notes.

---

## Collaboration areas

Relevant discussions can include:

- Web Audio review;
- music-software QA;
- Android beta testing;
- plugin listening review;
- release-readiness review;
- documentation cleanup;
- product positioning;
- technical partnership;
- mission work;
- funding or acquisition discussion.
