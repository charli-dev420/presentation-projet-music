## English

### How To Read This Page

This page gathers the public evidence signals that make the music projects understandable from the outside. It is not a raw log dump. It explains which product claims already have written support and which surfaces need a fresh review before stronger wording.

### DAW Core Evidence

**Project continuity.** DAW Core has a documented `.dw` direction covering project state, assets, snapshots, checksums, reload behavior, and re-export expectations. This supports the core product idea: a browser DAW should preserve musical work across a real round trip.

**Browser/desktop signal.** A desktop gate summary from 2026-06-23 is referenced as 25/25 with `findings=0` and virtual keyboard passing. That is useful as a dated development signal and should be refreshed when a new release statement is needed.

**Android beta.** Android is framed as a real test track for DAW Core browser workflows on devices and audio routes. The useful next evidence is a current device/workflow report for selected phones, browsers, and audio outputs.

### Unicor SoundEngine Evidence

**Synthé.** The seven-instrument Synthé family is documented with shared product behavior, manuals, visual material, and automated technical QA signals. The next valuable evidence is human listening review by selected instruments.

**FX.** FX families are grouped by musical treatment instead of being presented as isolated experiments. The next useful work is to prioritize which families deserve product polish first.

**VST distribution.** The catalog side has public visual material, manual references, and asset inventory signals. The next useful evidence is a clean review of catalog clarity, package readiness, support wording, and version alignment.

### What This Makes Credible

- DAW Core has a clear product center: browser-based web local-first DAW.
- The `.dw` direction is concrete enough to test through scenarios.
- Android beta work can be discussed through device and workflow evidence.
- Synthé, FX, and VST distribution are separate Unicor SoundEngine surfaces.
- The repo gives readers enough structure to decide where to test, help, invest, hire, or collaborate.
