## English

### Tutorial 1 - Understand The Repo In 15 Minutes

Read [overview](overview.md), then [DAW Core](daw-core.md), then [Unicor SoundEngine](unicor-soundengine.md). The result should be simple: DAW Core is the browser-based web local-first DAW; Unicor SoundEngine is the separate Synthé / FX / VST line.

Then open [project map](project-map.md) to see the project families and [resources](resources.md) to choose the next page.

### Tutorial 2 - Read DAW Core Like A Product

Start from the core scene: open the browser, create a project, save it, reopen it, listen, inspect the state, and continue editing. Read [user flows](user-flows.md), [evidence](evidence.md), and [QA validation](qa-validation.md) with that scene in mind.

The expected outcome is that you can explain why `.dw` continuity and Android beta testing matter.

### Tutorial 3 - Review A DAW Core Test Report

Name the browser or device, the audio route, the project scenario, the expected result, the observed result, reproducibility, and impact. Keep the language musical and practical: what did the user try to do, and did the project help them keep working?

### Tutorial 4 - Read Synthé, FX, And VST Distribution

Read [Unicor SoundEngine](unicor-soundengine.md), [synth suite](synth-suite.md), and [VST distribution](vst-distribution.md). Start from a user need: playable instrument, rhythm layer, tonal color, effect treatment, catalog clarity, manual quality, or package readiness.

The outcome should be a clear product note, not a list of repositories.

### Tutorial 5 - Prepare A Collaboration Note

Choose one angle: Android testing, web audio, local-first workflow, Synthé listening, FX prioritization, VST packaging, documentation, funding, mission, or role. Link the relevant page, state the concrete next step, and name the expected output.

## Francais

### Tutoriel 1 - Comprendre Le Repo En 15 Minutes

Lire [overview](overview.md), puis [DAW Core](daw-core.md), puis [Unicor SoundEngine](unicor-soundengine.md). Le resultat doit etre simple: DAW Core est le web local-first DAW dans le navigateur; Unicor SoundEngine est la ligne Synthé / FX / VST separee.

Ouvrir ensuite [project map](project-map.md) pour voir les familles projet et [resources](resources.md) pour choisir la page suivante.

### Tutoriel 2 - Lire DAW Core Comme Un Produit

Partir de la scene coeur: ouvrir le navigateur, creer un projet, sauvegarder, rouvrir, ecouter, inspecter l'etat et continuer l'edition. Lire [user flows](user-flows.md), [evidence](evidence.md) et [QA validation](qa-validation.md) avec cette scene en tete.

La sortie attendue est de pouvoir expliquer pourquoi la continuite `.dw` et la beta Android comptent.

### Tutoriel 3 - Lire Un Rapport De Test DAW Core

Nommer le navigateur ou l'appareil, la route audio, le scenario projet, le resultat attendu, le resultat observe, la reproductibilite et l'impact. Garder un langage musical et pratique: qu'est-ce que l'utilisateur a essaye de faire, et est-ce que le projet l'a aide a continuer ?

### Tutoriel 4 - Lire Synthé, FX Et Distribution VST

Lire [Unicor SoundEngine](unicor-soundengine.md), [synth suite](synth-suite.md) et [VST distribution](vst-distribution.md). Partir d'un besoin utilisateur: instrument jouable, couche rythmique, couleur tonale, traitement effet, clarte catalogue, qualite manuel ou readiness package.

La sortie doit etre une note produit claire, pas une liste de repositories.

### Tutoriel 5 - Preparer Une Note De Collaboration

Choisir un angle: tests Android, web audio, workflow local-first, ecoute Synthé, priorisation FX, packaging VST, documentation, financement, mission ou poste. Lier la page pertinente, formuler la prochaine etape concrete et nommer la sortie attendue.
