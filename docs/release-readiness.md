# Release Readiness / Preparation release

[EN](#english) | [FR](#francais)

## English

### Readiness Position

Release readiness is written as a set of product checks. DAW Core should only receive stronger availability wording when the target surface has a current validation summary. Unicor SoundEngine should only receive stronger product wording when the relevant instrument, FX family, catalog page, or package path has been reviewed.

### DAW Core Readiness

DAW Core readiness centers on the web local-first DAW workflow:

- create a project in the browser;
- save through the `.dw` direction;
- reopen on the target surface;
- confirm musical state, assets, snapshots, and playback;
- write a result that another person can understand.

Android readiness is a beta path until selected devices and audio routes have current reports.

### Unicor SoundEngine Readiness

Synthé readiness needs listening notes by instrument, preset review, manual clarity, and a clear first-session path.

FX readiness needs prioritization by treatment family and musical value.

VST distribution readiness needs catalog alignment, manual references, package notes, support wording, and visual consistency.

### Publication Standard

A readiness statement should be dated, scoped, and tied to the product surface. If the evidence is old, the wording should say less. If the scenario was rerun, the wording can say more.

## Francais

### Position Readiness

La readiness release est ecrite comme un ensemble de checks produit. DAW Core ne doit recevoir une formulation de disponibilite plus forte que si la surface cible a un summary validation courant. Unicor SoundEngine ne doit recevoir une formulation produit plus forte que si l'instrument, la famille FX, la page catalogue ou le chemin package concerne a ete revu.

### Readiness DAW Core

La readiness DAW Core se concentre sur le workflow web local-first DAW:

- creer un projet dans le navigateur;
- sauvegarder dans la direction `.dw`;
- rouvrir sur la surface cible;
- confirmer etat musical, assets, snapshots et playback;
- ecrire un resultat comprehensible par une autre personne.

La readiness Android reste une piste beta tant que les appareils et routes audio selectionnes n'ont pas de rapports courants.

### Readiness Unicor SoundEngine

La readiness Synthé demande notes d'ecoute par instrument, revue presets, clarte manuel et chemin premiere session clair.

La readiness FX demande priorisation par famille de traitement et valeur musicale.

La readiness distribution VST demande alignement catalogue, references manuels, notes package, wording support et coherence visuelle.

### Standard De Publication

Une formulation readiness doit etre datee, cadree et liee a la surface produit. Si la preuve est ancienne, le wording doit dire moins. Si le scenario a ete rejoue, le wording peut dire plus.
