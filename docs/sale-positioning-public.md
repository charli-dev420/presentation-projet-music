<h2 id="english">🇬🇧 English</h2>

This page keeps the public sale positioning readable without exposing buyer-only material.

It is not a pricing sheet, negotiation brief or dataroom index. It is a clean public framing for first contact, repo visitors and early evaluation.

---

## Short positioning

DAWWW-CORE is a local-first music production workspace delivered as a web desktop app with an Android Capacitor target and a portable `.dw` project container.

The project is best presented as a working music-studio codebase with a strong local-first foundation, not as a speculative prototype.

The commercial story is the combination of:

- visible product surface;
- portable project format;
- Web Audio based DAW direction;
- Android target;
- export and recovery workflows;
- account/billing access layers;
- a clear buyer-side handover and reconfiguration path.

---

## Buyer-facing value

DAWWW-CORE can be interesting as a technical asset to acquire, continue, harden, integrate or relaunch.

The buyer-facing value comes from the fact that several product layers already exist in the same codebase:

- DAW surfaces: arranger, sequencer, piano roll, mixer;
- audio features: instruments, effects, sampler, automation, modulation and export;
- portability: `.dw` project container and local-first fallback;
- product access: account/billing integrations;
- platform direction: browser desktop and Android Capacitor target;
- proof posture: QA, security, provenance, dependency and release-readiness material.

---

## How I want to frame it

I want to present DAWWW-CORE as:

- a serious music-production project;
- a local-first DAW/workspace codebase;
- a project with real surfaces and product structure;
- an asset that can be stabilized, continued or integrated;
- a project where the source can remain private while public explanation stays readable.

I do not want to present it as:

- a finished universal Android production release;
- a native VST host;
- a cloud-sync platform;
- a marketplace;
- a product where third-party accounts transfer automatically;
- a project with unlimited proof for every claim in the public repo.

---

## Safe public wording

Implemented local-first DAW/workspace codebase with portable `.dw` project format, web desktop app, Android Capacitor target, audio export, sampler, mixer, effects, automation, billing/account integrations and documented security/provenance controls.

Buyer-side reconfiguration is required for third-party accounts, stores, credentials and production release proofs.

---

## Public pack vs buyer pack

The public pack should support first contact:

- short sales deck;
- one-pager;
- selected screenshots;
- selected visual assets;
- lightweight technical appendix;
- public repo links.

The buyer pack should support due diligence:

- exact proof commands;
- claims policy;
- technical proof index;
- QA evidence;
- Android proof boundaries;
- billing/store/account transfer notes;
- security, provenance and dependency material;
- known limits and reruns;
- handover checklist.

Pricing, negotiation limits, private transfer notes and sensitive operational details should stay out of this public repository.
