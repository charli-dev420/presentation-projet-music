<h2 id="english">🇬🇧 English</h2>

This FAQ answers the main questions a public reader may have about DAWWW-CORE, UnicorSoundEngine and the scope of this repository.

---

## Is DAWWW-CORE open source?

No.

This repository is a public showcase and documentation entry point. It does not publish the private DAWWW-CORE source code, unreleased binaries, credentials, signing material, private proof artifacts or handover material.

The public repository is meant to make the project understandable without exposing sensitive material.

---

## What is DAWWW-CORE?

DAWWW-CORE is a local-first browser DAW / music workspace.

It is built around Web Audio, portable `.dw` projects, audio export, recovery workflows and an Android direction through Capacitor.

The central workflow is:

```text
create → save → reopen → recover → export → continue
```

The project is presented as a browser-based music-production system focused on project continuity.

---

## Does DAWWW-CORE load VST plugins?

No.

DAWWW-CORE does not load native VST plugins in the browser.

The instruments and effects inside DAWWW-CORE belong to the browser / Web Audio path.

Native VST and plugin work belongs to the separate UnicorSoundEngine line.

---

## What is UnicorSoundEngine?

UnicorSoundEngine is the separate plugin and audio-product ecosystem.

It can include:

- VST instruments;
- VST effects;
- manuals;
- presets;
- audition material;
- packaging;
- product pages;
- store and download flows.

UnicorSoundEngine belongs to the same music-tech portfolio as DAWWW-CORE, but it is not the DAWWW-CORE runtime.

---

## What is the difference between DAWWW-CORE and UnicorSoundEngine?

| DAWWW-CORE | UnicorSoundEngine |
| --- | --- |
| Browser DAW / local-first music workspace. | Separate VST / plugin ecosystem. |
| Web Audio runtime in the browser. | Native plugin products and audio-product packaging. |
| Portable `.dw` project workflow. | Instruments, effects, manuals, presets and distribution. |
| Does not host native VST plugins. | Covers the plugin product line. |

The separation is important because DAWWW-CORE should not be presented as a native VST host.

---

## What is the `.dw` format?

`.dw` is the project format associated with DAWWW-CORE.

Its role is to support project continuity: saving, reopening, recovering, moving and reviewing music sessions without treating the browser state as temporary and disposable.

In the public presentation, `.dw` is the center of the local-first project story.

---

## What is public in this repository?

This repository can publicly show:

- project positioning;
- documentation;
- selected visuals;
- public proof boundaries;
- safe technical summaries;
- public pack structure;
- useful review needs;
- links to readable project material.

It should stay understandable, safe and short.

---

## What remains private?

The following material should not be published directly here:

- private source code;
- unreleased binaries;
- credentials or signing material;
- private QA outputs;
- raw proof artifacts;
- account-transfer notes;
- buyer handover material;
- sensitive security details;
- negotiation notes.

Those elements belong to a controlled private review or buyer proof pack.

---

## What is the buyer proof pack?

The buyer proof pack is controlled review material for a deeper technical or acquisition discussion.

It can contain detailed proof, limits, reruns, QA material, transfer notes, security/provenance information and handover elements.

The buyer proof pack is not meant to be published as-is in this repository.

---

## Is Android production-ready?

The public repository should not claim universal Android production readiness.

DAWWW-CORE has an Android direction through Capacitor, but production-level Android claims require fresh validation, current artifacts and device-specific proof.

Public wording should stay careful:

```text
Android direction through Capacitor
```

rather than:

```text
fully certified on every Android device
```

---

## Where is the proof?

The public repository can contain proof summaries, selected screenshots, short demos, dated notes and safe references.

Detailed proof remains controlled.

A useful public proof should be easy to understand from outside the project:

- `.dw` save / reopen summaries;
- export examples;
- screenshots;
- short audio examples;
- dated validation notes;
- known limits.

---

## What kind of feedback is useful?

Useful feedback is concrete.

Examples:

- test `.dw` save / reopen / recovery;
- check browser playback behavior;
- review export scenarios;
- test Android boundaries;
- listen to instruments or effects;
- review presets and manuals;
- identify unclear public wording;
- check whether a claim sounds too strong.

A useful report explains what was tested, what happened, what failed or felt unclear, and what should be checked next.

---

## What should not be assumed from this repository?

This repository should not be read as proof that:

- DAWWW-CORE is open source;
- DAWWW-CORE hosts native VST plugins;
- every Android device is production-certified;
- third-party accounts or stores can be transferred automatically;
- all private proof material is public;
- every instrument or effect is fully release-certified.

The public repository is a showcase and documentation layer, not the complete private project archive.
