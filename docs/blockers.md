# Readiness Notes / Notes de diligence

[EN](#english) | [FR](#francais)

## English

These notes frame the next evidence work without turning the showcase into a blocker list. They are useful for partners, buyers, funders, testers, and mission leads because each item names the public value of closing the gap.

| Readiness item | Priority | Why it matters | Expected output |
| --- | --- | --- | --- |
| Release wording | P0 | Public claims should match current gate evidence. | Short desktop/Android release summary written for the target decision. |
| Android beta recertification | P0 | Android value increases when device proof is current. | `cert:android:studio-grade` style summary for selected devices/routes. |
| Cross-surface `.dw` proof | P1 | Project continuity is the main DAW Core value. | Replayed `.dw` roundtrip summarized in public-safe language. |
| Android device family coverage | P1 | Broader coverage makes beta feedback more useful. | Samsung/low-end/tablet expansion after the target RC or beta slice. |
| Studio accessibility | P2 | Repeat use improves when keyboard, labels, focus, and contrast are reviewed. | Focused accessibility pass tied to DAW Core studio surfaces. |
| Public asset hygiene | P2 | The showcase should stay light, valid, and trustworthy. | Reference audit before removing or replacing assets. |

### How To Use These Notes

Do not read this file as a failure report. Read it as a decision queue: each row can become a funded task, QA request, partner review, or demo prerequisite.
