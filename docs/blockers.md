# Readiness Notes / Notes de readiness

[EN](#english) | [FR](#francais)

## English

### How To Read These Notes

These notes are a work queue, not a warning label. Each item describes the kind of polish or validation that would make the public project presentation stronger.

### Priority Work

**DAW Core Android beta.** Run current browser/mobile scenarios on selected Android devices, with the device class, browser, audio route, scenario, result, and impact written clearly.

**`.dw` continuity replay.** Keep one small project scenario ready for review: create, save, reopen, check assets and snapshots, continue editing, then write the result.

**Release wording refresh.** When a release or availability statement is needed, tie it to the current validation summary instead of relying on older proof language.

**Synthé listening pass.** Pick the instruments closest to release usefulness and write listening notes around role, presets, controls, problems, and first-session clarity.

**VST catalog polish.** Check that names, visuals, manuals, product pages, and packaging notes tell the same story.

**Documentation upkeep.** Keep the README, overview, DAW Core page, Unicor SoundEngine page, evidence, and resources aligned after each meaningful project change.

## Francais

### Comment Lire Ces Notes

Ces notes sont une file de travail, pas une etiquette d'alerte. Chaque point decrit le polish ou la validation qui rendrait la presentation publique plus forte.

### Travail Prioritaire

**Beta Android DAW Core.** Lancer des scenarios navigateur/mobile courants sur appareils Android choisis, avec classe d'appareil, navigateur, route audio, scenario, resultat et impact ecrits clairement.

**Replay de continuite `.dw`.** Garder un petit scenario projet pret pour revue: creer, sauvegarder, rouvrir, verifier assets et snapshots, continuer l'edition, puis ecrire le resultat.

**Rafraichissement wording release.** Quand une formulation release ou disponibilite est necessaire, la lier au summary validation courant au lieu de reutiliser un langage de preuve ancien.

**Passe d'ecoute Synthé.** Choisir les instruments les plus proches d'une utilite release et ecrire des notes d'ecoute autour du role, presets, controles, problemes et clarte premiere session.

**Polish catalogue VST.** Verifier que noms, visuels, manuels, pages produit et notes packaging racontent la meme histoire.

**Maintenance documentation.** Garder README, overview, page DAW Core, page Unicor SoundEngine, evidence et ressources alignes apres chaque changement projet significatif.
